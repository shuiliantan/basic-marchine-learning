迁移学习：把从任务A中学习到的信息迁移到另外一个不同的任务B中的方式。要求

  • 任务A和任务B有相同的输入；
  • 必须任务A比任务B有更多的数据。(如果任务A的数据比任务B少时，并不能得到很大的增益。)

训练方式

  • B的数据量小时：可以只训练最后一层和输出层的神经网络，或者训练最后一到两层的神经网络；

  • B的数据量大时：可以对神经网络的所有参数都进行重新训练

    ​ 此时训练的初始化阶段称为预训练（pre-training），因为使用A的数据预初始化/预训练了神经网络的权重。在B上的训练有时被称为微调（fine_tuning）

example

  1. 图像识别：有100万张图像检测的任务A，和100张放射扫描诊断的任务B。在A上预训练可以学习到很多图像的组织节后和自然特性信息，比如边界检测、曲线检测、明暗对象检测等。能帮助在任务B中学习的更快，或者减少一些学习的数据。
  2. 语音检测：训练好的语音识别系统A（输入为语音，输出对应的文字），和‘唤醒词’/‘触发词’检测系统，比如用hisiri，唤醒苹果系统。可以创建好几层新的神经网络，来给唤醒系统打上新的Y标签

目标：检测移动app上，用户上传的照片是不是猫的照片

数据集：20万张web爬下来的照片（分辨率高的）、1万张移动app照片（拍摄和构图不怎么专业，大多是糊的）

切分方式1：20万张web照片和1万张移动app照片随机混合，然后切分

  • 优点：训练集验证集测试集都是同一个分布，
  • 缺点：设置的开发集使团队致力于优化1个与实际关心的目标 并不相同的数据分布

切分方式2：训练集=20万个web照片+5000个app照片，验证集和测试集分别2500个app照片

  • 优点：长期来说性能更好
  • 缺点：训练集和验证集/测试集来自不同分布

![image-20200525095512551](/Users/shuiliantan/Library/Application Support/typora-user-images/image-20200525095512551.png)

切分的目的：

  • 建议dev 和test set 来自同一分布，因为这样你们就瞄准了目标

  • 现在的趋势是 使用更多数据作为训练集 使用较少的数据作为开发集和测试。

  • 测试集：作用是帮我们评估最终系统的性能，因此，测试的大小只要能保证对系统整体性能评估的高置信度即可。

    有时候没有测试集也是可以的，不过不建议这么做，最好在上线前使用测试集得到对系统性能的无偏估计。

当用户想要自定义切分，可能训练集和验证集不来自同一分布时，对于长期的目标是好的。我们作为平台方没办法控制

用户的需求是没办法去定义的。本来数据是混合在一起，作为平台上只可能尽可能的保证训练集这验证集来自统一分布，但是用户只关心app上的照片检测的准确率，所以

Q : 当发现标签错误时，值得花费时间取修正这些标签吗？

A：深度学习算法对训练集的随机错误很稳健，只要标记的错误不是那么偏离随机分布，比如只是由于人的偶然错误造成的。如果这些错误是很随机的错误，基本上可以不管，不用花费太多时间取修正。当然，纠正也是没问题，有时候也是很有必要的。但是只要总数据量够大，实际错误标记的label占比不高，也可以不做。

但是深度学习算法对系统错误不稳健，例如：如果一直把狗标记为猫，那么算法最后就会把狗预测为猫

![image-20200522095447703](/Users/shuiliantan/Library/Application Support/typora-user-images/image-20200522095447703.png)

建议：

  • dev test set same distribution

  • 对right和wrong的预测都需要check观察。理解机器犯什么错，探究数据，计算错误率，这样可指导下一步去做什么

  • Train slightly different distribbution from dev/test set
